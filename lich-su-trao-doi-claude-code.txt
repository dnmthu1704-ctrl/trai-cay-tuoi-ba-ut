6. NÂNG CẤP: THÊM BACKEND + DATABASE THẬT (SUPABASE)
----------------------------------------------------------------
- User đồng ý nâng cấp theo đúng kiến thức buổi 2: dùng Supabase (kết
  hợp Database + Data API tự sinh đóng vai trò Backend).
- Các bước thực hiện:
  + User tự tạo project Supabase (vì cần đăng nhập tài khoản cá nhân).
  + Claude Code soạn SQL tạo bảng "orders" (ho_ten, so_dien_thoai,
    loai_hop, so_luong, dia_chi, ghi_chu) + RLS policy cho phép insert.
  + User chạy SQL trong Supabase SQL Editor.
  + Debug nhiều vòng lỗi 401 "row-level security policy violation":
    - Lỗi 1: key publishable mới không khớp role anon trong policy cũ
      -> sửa policy "to public".
    - Lỗi 2: role anon thiếu GRANT INSERT cấp bảng -> thêm GRANT.
    - Lỗi 3 (nguyên nhân thật): câu lệnh TEST của Claude dùng header
      "Prefer: return=representation" yêu cầu trả lại dòng vừa insert,
      nhưng bảng chưa có policy SELECT -> Postgres báo lỗi RLS dù
      INSERT thực ra hợp lệ. Đổi sang "return=minimal" (đúng như code
      thực tế trong script.js) -> insert thành công (HTTP 201).
  + Cập nhật script.js: gửi dữ liệu form qua fetch() POST tới Supabase
    REST API (https://lpohquwgcptxaeydxoqp.supabase.co/rest/v1/orders).
  + Commit + push code, dọn dữ liệu test trên Supabase.
  + Chuyển repo từ private sang public (gh repo edit --visibility
    public), bật lại GitHub Pages (đã bị tắt tự động khi repo từng ở
    chế độ private).

KIẾN TRÚC CUỐI CÙNG CỦA SẢN PHẨM
================================================================
- Frontend: index.html, style.css, script.js (giao diện, banner/popup
  khuyến mãi, form đặt hàng, chọn trái cây mix)
- Backend: Supabase Data API (PostgREST tự sinh từ schema database) +
  1 Postgres function (has_used_promo) xử lý logic kiểm tra khuyến mãi
- Database: Supabase PostgreSQL, bảng "orders" (lưu đơn hàng thật của
  khách), có Row Level Security (RLS) bảo vệ dữ liệu, GRANT quyền insert
  cho role anon
- Deployment: GitHub Pages (build từ branch main)
- Bảo mật: dùng anon public key (được phép public) cho phía client,
  RLS + RPC giới hạn quyền truy cập dữ liệu, không có API key/secret
  nào bị lộ trong code.
